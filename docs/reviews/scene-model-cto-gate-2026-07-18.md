# Scene-Model Graft — Production Gate Review (SideFX-CTO lens)

**Date:** 2026-07-18
**Reviewer:** SIDEFX-CTO lens (production gate, per Joe's order 2026-07-18)
**Subject:** Mile-2 synthesis blueprint (`scratchpad/mile2/synthesis.json`, winner A-minimal + 15 grafts) — graft Cognitive-Bridge's assertion/resolution engine into SYNAPSE as a standing scene-model.
**Standing law honored:** product readiness only. Patent-relevant facts appear solely in the non-blocking appendix.

---

## Executive summary

**Verdict: ALLOW-WITH-CONDITIONS (10 conditions, numbered and testable, below).** The blueprint is unusually well-grounded for a feature at this stage — instrumentation-first (Mile 0), a pre-substrate falsifier (Mile 0.5), default-OFF staged flags, two measured kill points, and every hook seam verified against the real code by an adversarial critic. The premise is validated by SYNAPSE's own measurements (~22% MCP-path re-observation, ~8–9s median turn). But the review found three structural defects in the confidently-wrong-model class that every design, every judge, and the synthesis missed, all verified against the repo: **(1)** undo — SYNAPSE registers `undo`/`redo` as mutating commands, its router actively *recommends* `houdini_undo` for rollback, and the product's core guarantee is one-Ctrl+Z reversibility, yet no Mile-1/Mile-2 artifact contains the word "undo" as a scene-change source; the natural implementation (no-op mapper listing) leaves the store confidently wrong *and* refreshes its staleness baseline so the wrongness presents as fresh; **(2)** the staleness comparator as specced cannot work on the injection path — the stored `hash_after` is `None` for essentially all panel and `/mcp` mutations (envelope hash capture is TLS-suppressed for bridge-routed ops, `handlers.py:414/:425`), and stored vs fresh hashes are target-scoped to *different subtrees*, so the banner is either permanently UNKNOWN (steering line self-defeating) or, if an implementer quietly compensates, silently blind; the hash is also cookCount-based and therefore structurally blind to non-cooking parm edits — lights, materials, OBJ transforms — the most common artist edit class; **(3)** the mid-send window — the volatile block is frozen at worker `__init__` across a ≤25-iteration send while the C5 concurrent-transport scenario (and the artist) can mutate underneath it, with the per-iteration conflict check parked to v2. Each defect has a bounded fix that slots into the existing mile plan without redesign; Miles 0 and 0.5 are unaffected and can start immediately. With the conditions below, this ships as an honest, measured, killable experiment — which is exactly what the blueprint says it wants to be.

## Verdict

**ALLOW-WITH-CONDITIONS.** Miles 0 and 0.5 may start now. Conditions 1, 5, 9 block Mile 2. Conditions 2, 3, 4 block Mile 3. Conditions 6 blocks Mile 0.5/5 sign-off. Conditions 7, 8 block the default-ON decision. Condition 10 blocks nothing here but must land in the H22 boundary-wiring checklist.

### Conditions (each testable)

1. **Undo/redo mapper = whole-store invalidation (blocks Mile 2).** `undo` and `redo` are registered mutating commands (`python/synapse/server/handlers.py:713-714`, absent from `_READ_ONLY_COMMANDS` :190-220) and the router recommends `houdini_undo` for rollback (`python/synapse/routing/router.py:425,:491`; `panel/hda_controller.py:174` sends `undo` directly). They MUST have a mapper entry that clears the store or marks every fact stale and increments the dirty counter — never the explicit no-op list. **Test:** the mapper-coverage conformance test asserts `undo`/`redo` are NOT on the no-op list; a store-level test drives cmd_type `undo` through the hook and asserts zero facts survive as "known".
2. **Artist-undo detection via the undo-stack evidence APIs (blocks Mile 3).** Add an undo-stack drift probe to the injection-time staleness check using `hou.undos.undoLabels()`/`redoLabels()` — already verified in production use by the bridge's evidence-derived undo check (`shared/bridge.py:180,:1010`). Snapshot `(len(undoLabels), len(redoLabels), top-label)` at each store write; at injection time, redo-stack growth or top-label regression ⇒ staleness banner + dirty. **Test:** scripted `hou.undos.performUndo()` between two sends flips the banner ON.
3. **Replace the staleness comparator (blocks Mile 3).** The specced "last `hash_after` vs fresh `capture_scene_hash`" is unsound twice over, verified: (a) `_hash_before/_hash_after` are captured only when `_enveloped` is true (`handlers.py:414,:425`) and `envelope_active()` is False under the `bridge_routed` TLS for all panel + `/mcp` bridge-routed mutations (critic-established, `integrity_envelope.py:107-144`) — so the store's hash baseline is `None` on exactly the injection path; (b) each capture is scoped by `_hash_target(payload)` to the *op's own* subtree (`integrity_envelope.py:150-159` → `shared/bridge.py:568-614`), so stored and fresh hashes describe different subtrees. Replace with: at injection time, capture over a **fixed root set** (`/obj`, `/stage`, `/mat`, `/out`) on the panel main thread and compare against the *previous injection-time* capture of the same roots; `None` on either side ⇒ UNKNOWN banner. Additionally, because the hash sees parm changes only via `cookCount` (`shared/bridge.py:600-604`) and is blind to non-cooking parm edits (lights, materials, OBJ transforms) and to node renames (no rename command exists — verified against the registry — and rename changes neither `sessionId` nor `cookCount` [INFERENCE — probe: rename a node, compare hashes]), the design doc MUST state that **hash-equal is not freshness proof for parm-level facts**, and the renderer must label parm facts with their as-of op index rather than implying verification. **Test:** unit pins the fixed-root capture pair; a light-intensity hand-edit test documents the known blindness (expected-fail marker, so the limitation is pinned, not hidden).
4. **Mid-send external-mutation guard (blocks Mile 3).** The volatile block is fixed at worker `__init__` (by design, C6) across a ≤25-iteration send. The store is process-global and already sees cross-transport writes, so: record a send-scoped `external_mutation_during_send` flag when any write arrives from a transport other than the active send's; the Mile-4 pull tool surfaces it immediately; the Mile-5 report MUST publish how often it fired (this sizes the residual risk of the parked v2 veto). The injected block's as-of timestamp line gains: "changes after this timestamp are not reflected." **Test:** store unit — write from a second fake transport mid-send sets the flag; report unit counts it.
5. **No-op list is justified line-by-line (blocks Mile 2).** Every cmd_type placed on the mapper no-op list carries a one-line justification in the conformance test file. Specifically: `cops_temporal_analysis` (moves the global playhead — `handlers.py:217-219`), the `tops_*`/`hda_*` cook-and-mutate commands, and any handler whose result can indicate an in-handler `performUndo` rollback (`handlers.py:1115-1137`, `handlers_solaris_graph.py:489`, `handlers_cops.py:378,:884`, `handlers_hda.py:592`) must map to dirty-or-facts, never silent no-op; results carrying rollback/partial-failure markers map to dirty, and `batch_commands` fan-out skips failed sub-ops. **Test:** conformance test fails on an unjustified no-op entry; batch fan-out test includes a failed sub-op.
6. **Experiment hardening (blocks Mile 0.5 sign-off and the Mile 5 verdict).** (a) Mile 0.5 spike: ≥2 task shapes, one mirrored from a real high-re-read transcript (e.g. session `d102983f`, 54% re-read), ≥3 runs per arm per task (LLM nondeterminism — a single run is a coin flip), and the fake block byte-format-identical to the Mile-3 renderer spec including the steering line. (b) Mile 5: pre-registered flag alternation (not ad-hoc flipping), sessions stratified observation-heavy vs execute-heavy, cluster-level dispersion reported (10 organic sessions/arm cannot resolve a 5pp delta against 0–54% per-session variance — the scripted matched-task battery is the *primary* evidence, organic corroborating), and provider **model id** recorded per session in the U2 ledger (a mid-window model update is a confound). **Test:** report script refuses a verdict when stratification metadata is absent.
7. **Cache-not-memory declaration (blocks default-ON).** The v1 store is declared ephemeral and non-authoritative in the semantics RFC and VENDOR.md: in-memory only, cleared on AfterLoad/hip-change, never persisted, never citable as provenance. Audit, FloorGate, and the envelope trail remain the truth surfaces; `agent.usd` remains the receipts ledger named by the ratified boundary contract (`docs/SYNAPSE_H22_BOUNDARY.md` §8 `provenance_not_bypassed`), reachable later only via RFC D-3. This caps the store-fragmentation trajectory (jsonl + Moneta + markdown scene memory + dormant agent.usd + RAG) at "plus one cache," not "plus one memory system." **Test:** a lint/grep conformance check that the scene-model module has no file-write path in v1.
8. **Support-surface consolidation (blocks default-ON).** (a) `SYNAPSE_READ_LEDGER` default-ON writes one JSONL row per read command through the shared 2-worker `_log_executor` (`handlers.py:72`) that also carries audit + envelope writes — state the FIFO cap value, buffer appends (flush ≤1/s or every N rows), and document the `~/.synapse/logs` footprint for network-home installs in DEPLOYMENT.md. (b) `synapse_doctor`/status reports one triage line: effective flag value, store size, eviction count, staleness verdict, ImportError-degrade cause; the Mile-4 tool's "disabled" note names the cause (flag vs ABI). The four-value flag matrix is explainable to a support desk only through that single surface. **Test:** doctor output pinned by a unit test; ledger flush-batching unit.
9. **Dirty-counter reset semantics defined (blocks Mile 2).** Currently undefined — in execute_python-heavy sessions (43.3% of measured MCP calls) the banner would be permanently on, making injection permanent noise. Define: dirty resets only on whole-store clear (AfterLoad/hip-change/undo-invalidation) or when a fresh allowlisted observation re-covers the affected scope. **Test:** store unit pins reset behavior.
10. **APEX-MCP out-of-band mutations (rides the boundary work, not this gate).** When D-H22-1/2 wiring (`2.7`/`2.8`) lands, first-party APEX MCP tool calls that mutate the scene will not route through `SynapseHandler.handle()` and are invisible to the store [INFERENCE — confirmed by the planned `science/mcp_surface_probe.py` enumeration]. Add to the D-H22-4 checklist: APEX-MCP-driven mutations must either route through `handle()` or mark the scene model dirty. Rigging/APEX authoring itself stays a non-goal per the ratified boundary — the store maps SYNAPSE commands only.

## A · The confidently-wrong-model class

The gate question for every path: does the staleness banner + steering line degrade to *today's* behavior (model re-observes), or can it degrade *below* today (model acts on a stale fact it would previously have re-read)? Enumeration, tiered:

| # | Path | Visibility to the store | Verdict as blueprinted | Tier |
|---|------|------------------------|------------------------|------|
| 1 | Artist parm edit between sends (light intensity, material parm, OBJ transform — non-cooking nodes) | Invisible. Hash sees parms only via `cookCount` (`shared/bridge.py:600-604`); non-cooking edits never bump it. DriftConflict fires only if a seeded observation happens to re-cover the path. | **BELOW today.** Machine-authored `set_parm` echo presents as fresh; steering line says don't re-inspect. → Condition 3. | VERIFIED (hash impl read) |
| 2 | Artist node rename | Invisible. No rename cmd_type exists (registry grep), `sessionId`/`cookCount` unchanged on rename. Ghost facts stand under the dead path. | **BELOW today** — model addresses a nonexistent path; today it would inspect first. → Condition 3 labeling + the error itself recovers in one turn (bounded). | VERIFIED (no command) + INFERENCE (rename hash-invisibility; 1-line probe) |
| 3 | Artist Ctrl+Z of an agent op | No command fires. Create/delete reverts visible only if the (broken) comparator were sound; parm reverts invisible per #1. Detection primitive exists and is unused: `hou.undos.undoLabels/redoLabels`, already production-verified in `shared/bridge.py:180,:1010`. | **BELOW today**, on the single most common scene change in a product whose core guarantee is one-Ctrl+Z reversibility. → Condition 2. | VERIFIED |
| 4 | Agent-driven `undo`/`redo` commands | Fully visible at the choke point (`handlers.py:713-714`, mutating-classified) — but **no artifact in the entire evidence chain maps them** (grep across all 12 mile1/mile2 files: zero mentions as a scene-change source). ## B · Second-order / trajectory

- **Boundary contract (`docs/SYNAPSE_H22_BOUNDARY.md`, ratified 2026-06-27):** compatible. The store maps SYNAPSE's own COP/LOP/SOP/Karma/USD commands; no rigging drift, no corpus forking. One real tension: the ratified moat names `agent.usd` as the receipts ledger ("no ledger entry ⇒ it didn't happen") while its provenance writers stay dormant and this blueprint stands up a *parallel* scene-truth surface. That is defensible only if the store is loudly a cache, not a memory system — condition 7. The v2 persistence target should be `agent.usd` via RFC D-3, not a sixth sidecar.
- **APEX MCP direction:** complementary, with one blind spot. A first-party MCP mutating the scene out-of-band is a new C5-class writer the blueprint's concurrency story never lists — condition 10 attaches it to the already-planned D-H22-4 surface probe. The scene model does not compete with the vendor's fluency surface; it is client-side dedup of SYNAPSE's own observations.
- **Declarative pressure:** injecting facts does **not** reduce the pressure to extend declarative coverage — the waste classes are disjoint (declarative kills O(N) *mutation* turns; the scene model kills *re-observation* turns, the measured 22.4%). If anything they compound: propose/instantiate results are the cleanest structured inputs the mapper will ever see. The real cost is opportunity cost (5–8 days vs COPs/TOPS declarative work), priced honestly by the judges' benefit-per-week scoring.
- **Memory fragmentation:** the honest count today is jsonl (default) + Moneta (opt-in) + markdown scene memory + dormant agent.usd + the RAG corpus. The seam report's ruling is correct — the resolver does not fit behind the episodic `MemoryStore` seam — so a sibling substrate is the right call *only* under the cache declaration (condition 7). Five memory systems is a trajectory problem; five memory systems plus one ephemeral cache is not.
- **Substrate bet:** vendoring CB in degenerate last-writer-wins mode carries dormant LIVRPS machinery (the maintainer's YAGNI objection stands), but the kill criterion prices it: if Mile 5 nulls, the vendored tree parks as paid-down v2 infrastructure and the instrumentation remains. Acceptable.

## C · Support burden at seat scale

- **Read ledger (default ON):** the only *default-on* behavior change in the whole blueprint, and therefore the first thing a support desk hears about. One JSONL append per read command through the shared 2-worker `_log_executor` (`handlers.py:72`) that also carries audit + envelope writes; on network-redirected home dirs every append is a round trip. Cap value unstated in the blueprint. Condition 8a (buffering, stated cap, DEPLOYMENT.md note).
- **Flag matrix:** `SYNAPSE_SCENE_MODEL` off|write|inject|full × `SYNAPSE_READ_LEDGER` × `SYNAPSE_SCENE_MODEL_TOKENS` is a support-desk decision tree unless there is exactly one triage surface — condition 8b (doctor line).
- **Tri-interpreter pydantic matrix:** the honest fragility. `_vendor` ships cp311/cp313 only; the graft makes pydantic load-bearing for scene truth. The Mile-1 tri-interpreter conformance gate + loud-degrade-via-doctor is the right posture; the artist-facing symptom of a future Houdini Python bump is "the feature silently vanished," which conditions 8b's cause-naming makes diagnosable. The per-Houdini-build re-vendor tail is real, bounded, and already an accepted cost of the `_vendor` mechanism.
- **Eviction:** cap 2000 paths / 1000 events with the counter surfaced in the render header — visible degradation, correctly designed.
- **Net-new artist-reportable failures:** wrong action from a stale fact (§A — conditioned), permanent stale banner (condition 9), home-dir disk growth (condition 8a), post-build-bump silent no-op (condition 8b). All four have named mitigations; none survives the conditions unmitigated.

## D · The experiment design

- **Mile 0.5 spike:** right instinct (cheapest falsifier, pivot-not-kill severity is correctly calibrated), under-specified as written: one scripted task, no repetition count, fake-block format unspecified. A null on a single run of a single task falsifies nothing — LLM nondeterminism alone produces that. Condition 6a.
- **Mile 5 A/B:** ≥20 organic sessions is directional, not decisive. Per-session re-read rates span 0–54%; with ~10 sessions/arm the cluster-level variance dwarfs a 5pp effect. The blueprint already contains its own rescue — the scripted matched-task battery — but ranks it second. Invert: scripted battery primary, organic corroborating, stratified, pre-registered assignment, model-id logged. Condition 6b.
- **The ≥5pp bar:** on a 13.4% pure-re-read base this is a ~37% relative reduction — an ambitious, correct ship bar for a default-ON flip. Keep it.
- **The OBSERVED-present guard** (skeptic): genuinely good experiment design — it is the only thing making a null unambiguous. Keep verbatim.
- **Instrument symmetry:** ledger identical in both arms; injection token cost counted in the report. Sound.

## E · What everyone missed

1. **Undo, in all three forms** (§A #3–4, conditions 1–2) — the headline. Twelve artifacts, three designs, three judges, one synthesis, one adversarial critic: zero mentions of the product's most common scene change and its own recommended recovery command.
2. **The comparator's None-starvation on the injection path** (§A #7, condition 3) — the critic caught the C1 hook-placement trap (write hook must be unconditional) but nobody followed the same TLS-suppression fact one step further to the *hashes the hook receives*.
3. **Mid-send staleness window** (§A #8, condition 4) — the C6 cache protection and the staleness guarantee are in direct tension; frozen-at-`__init__` is cited as a feature everywhere and as an exposure nowhere.
4. **Dirty-counter reset semantics** (condition 9) — undefined lifecycle for the mechanism carrying the execute_python opacity story.
5. **APEX MCP as an out-of-band writer** (condition 10) — the C5 story lists SYNAPSE's own transports only, while the repo's own ratified boundary doc plans wiring a first-party mutation source this cycle.
6. **The store's persistence posture** (condition 7) — no artifact states that v1 never touches disk; the five-store fragmentation critique is only answerable with that sentence written down.
